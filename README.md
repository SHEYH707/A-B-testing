# A/B-тестирование для показателей кликабельности (CTR)

Локальное информационно-аналитическое приложение на Python для анализа A/B-экспериментов. Программа загружает CSV-файл, проверяет и очищает данные, рассчитывает CTR вариантов A и B, выполняет двусторонний z-тест двух долей, строит графики и сохраняет полный комплект результатов.

## Возможности

- загрузка и предпросмотр CSV-файла;
- проверка структуры, пропусков, категорий и типов данных;
- удаление несогласованных записей и повторных пользователей;
- расчёт общего и дневного CTR;
- описательные статистики дневного CTR;
- абсолютное и относительное сравнение вариантов;
- z-тест, p-value и 95%-й доверительный интервал;
- четыре графических отчёта;
- сохранение очищенных данных, таблиц, графиков и текстового вывода.

## Установка

Откройте Anaconda Prompt и перейдите в папку проекта:

```bash
cd путь_к_папке_A_B_tests
```


## Запуск

```bash

```

После запуска откроется графическое окно. Работа выполняется кнопками **«Загрузить датасет»**, **«Выполнить анализ»**, **«Построить графики»** и **«Сформировать отчёт»**.

## Формат входных данных

Одна строка CSV-файла соответствует одному показу варианта пользователю.

| Столбец | Допустимые значения | Описание |
|---|---|---|
| `user_id` | идентификатор | Пользователь эксперимента |
| `timestamp` | дата и время | Момент показа |
| `group` | `control`, `treatment` | Группа эксперимента |
| `landing_page` | `old_page`, `new_page` | Показанная страница |
| `converted` | `0`, `1` | Признак целевого действия |

Значение `converted = 1` считается целевым действием. Вариант A соответствует группе `control`, вариант B — группе `treatment`.

```text
CTR = clicks / impressions × 100%
```

Перед анализом удаляются строки с несогласованными парами `control`–`new_page` и `treatment`–`old_page`, а также повторные `user_id`.

## Порядок работы

1. Нажать **«Загрузить датасет»** и выбрать CSV-файл, например `data/ab_data.csv`.
2. Проверить предпросмотр данных.
3. Нажать **«Выполнить анализ»**.
4. Изучить статистический отчёт.
5. Открыть графики или сформировать итоговый отчёт.

## Статистическая проверка

Используется двусторонний z-тест для двух независимых долей:

- `H0`: CTR вариантов A и B одинаков;
- `H1`: CTR вариантов A и B различается;
- уровень значимости: `α = 0.05`.

При `p-value < 0.05` различие считается статистически значимым. Более высокий наблюдаемый CTR без статистической значимости не является доказанным преимуществом варианта.

## Графики

- столбчатая диаграмма общего CTR;
- гистограмма дневного CTR;
- диаграмма рассеяния CTR по датам;
- диаграмма Бокса–Вискера.

## Тестирование

```bash
python -m pytest -v
```

Тесты находятся в `tests/test_ab_analysis.py` и проверяют загрузку, валидацию, очистку, расчёт статистик, z-тест и формирование текстового отчёта.

## Источник данных

В проекте используется датасет [A/B Testing](https://www.kaggle.com/datasets/zhangluyuan/ab-testing). Тестовый файл расположен в `data/ab_data.csv`.

## Ограничения

- поддерживаются только две группы: `control` и `treatment`;
- статистическая проверка выполняется при фиксированном уровне значимости `0.05`;
- после очистки для каждого `user_id` сохраняется одна запись;
- приложение предназначено для локального запуска и не использует сетевую базу данных.
